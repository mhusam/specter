# GitHub discovery checklist (About, topics, Discussions)

This file lists steps **only you can do in the browser** (this environment has no `gh` / API access to your account). Completing them takes about two minutes and improves search and first impressions.

## 1. Repository “About”

On [github.com/mhusam/specter](https://github.com/mhusam/specter), click **⚙️** next to **About** → edit.

**Suggested description** (copy-paste):

```text
Local-first AI specs from plain ideas — Ollama, 22-doc spec package, 31-doc catalog. Docker + PostgreSQL + React. No cloud LLM.
```

**Suggested website** (optional): leave blank or add a demo/docs URL when you have one.

## 2. Topics

Still in About → **Topics** → add (one at a time or comma-separated where supported):

`ollama` · `local-ai` · `self-hosted` · `specifications` · `requirements` · `documentation` · `postgresql` · `docker` · `react` · `typescript` · `vite` · `express` · `sse` · `ai-agents` · `spec-driven-development` · `open-source`

Remove any that GitHub rejects (rare); keep at least **ollama**, **local-ai**, **self-hosted**, **documentation**.

## 3. Social preview

**Settings** → **General** → **Social preview** → **Edit** → upload an image (e.g. crop from `screenshots/home.png` to ~1200×600). Improves cards when the repo is shared on social sites.

## 4. Discussions

**Settings** → **General** → **Features** → enable **Discussions**.

Then the “Ask a question” shortcut in the issue chooser (`.github/ISSUE_TEMPLATE/config.yml`) will work.

## 5. Community standards

After `CODE_OF_CONDUCT.md` and `SECURITY.md` are on `main`, open **Insights** → **Community Standards** and complete any remaining suggestions (e.g. issue templates — you already have some under `.github/ISSUE_TEMPLATE/`).
